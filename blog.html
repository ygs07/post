Version Control: Checkpointing Your Work

Imagine you’re playing a game. You reach a checkpoint at specific points, and the game saves your progress. Even if you exit, turn off your console, or delete the game entirely, you can return later and pick up right where you left off.

Version Control works just like that. When you reach a point in your code or project where you feel: 

  • “Hmm, I’ve done enough,” or
  • "I’ve finished this feature,”
    you can use Version Control to save the current state of your code.

 

At any time, you can revisit earlier states or continue from where you stopped. Essentially, Version Control is about checkpointing your work.

 

What's GitHub?:

Think of GitHub as iCloud or Google Drive for your code. After creating checkpoints in your project, GitHub ensures your progress is securely stored and accessible anywhere.

For example:

  • Your PC crashes? GitHub has your code backed up.
  • Collaborating with others? GitHub makes it easy to share and sync code across teams, even if you work on different schedules.

GitHub stores your code in repositories. These can be:

  • Private: For personal projects or sensitive work.
  • Public: For open-source projects or sharing knowledge.

Other services, like Bitbucket and GitLab, offer similar functionality but GitHub remains the most popular.

 

The Power of Version Control

Version Control doesn’t just help individual developers—it’s how teams manage complex projects. Ever wondered how apps like Instagram can release updates while older versions still work? That’s Version Control in action. Each version of an app is like a checkpoint, representing significant progress or milestones in its development.

 

Let’s Talk About Git

Git is the language of Version Control. It’s how developers communicate with their code’s history. While there are alternatives to Git, it’s by far the most widely used.

Here’s a quick overview of some essential Git commands:

  • git init: Activates Git in your folder.
  • git branch <branch-name>: Creates a separate branch (or version) of your code, allowing you to work on a feature without affecting the main project.
  • git add: Selects specific files to include in your next checkpoint.
  • git commit: Records a checkpoint with a short description of the changes.
  • git push: Upload your code to GitHub (or other services like Bitbucket or GitLab).

All these commands are run in your terminal (Mac) or Command Prompt (Windows).

 

Or Use GitHub Desktop For Simplicity

If typing commands in the terminal feels intimidating, GitHub Desktop is a great alternative. It’s an app that simplifies the process:

  • Download GitHub Desktop.
  • Log in with your GitHub account.
  • Select your repository.
  • Start checkpointing your work—all without touching the terminal!
